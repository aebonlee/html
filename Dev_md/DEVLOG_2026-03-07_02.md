### 5. 모바일 2차 메뉴 줄바꿈 개선

**커밋**: `84fe0a1`

#### 문제
`.mobile-tabs-row`가 `overflow-x: auto` + `scrollbar-width: none`으로, 가로 스크롤 필요하지만 스크롤바가 숨겨져 사용자가 메뉴 존재를 인지할 수 없음.

#### 해결
```css
/* Before */
.mobile-tabs-row {
  display: flex;
  overflow-x: auto;
  scrollbar-width: none;
}

/* After */
.mobile-tabs-row {
  display: flex;
  flex-wrap: wrap;
  gap: 6px;
}
```

#### 추가 개선
- `.mobile-tab`: 테두리(`border: 1px solid`) 추가로 탭 구분 명확화
- `.mobile-chip`: `flex-shrink: 0` 제거, 패딩 축소 (`14px→12px`)
- 모든 탭/칩이 화면 내에서 줄바꿈되어 한눈에 보임

### 6. Navbar `fixed` → `sticky` 근본 정렬 해결

**커밋**: `456c0fc`

#### 문제 (근본 원인)
| 요소 | position | 너비 기준 |
|------|----------|----------|
| navbar | `fixed` | **뷰포트 전체** (스크롤바 포함) |
| container | `static` | **콘텐츠 영역** (스크롤바 제외) |

→ 스크롤바 너비(~17px)만큼 중앙 정렬 기준이 달라, 로고보다 카드가 왼쪽으로 ~8.5px 치우침. 오른쪽도 동일하게 어긋남.

#### 해결
```css
/* Before */
.navbar {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
}
.hero { margin-top: var(--navbar-height); }
.page-content { margin-top: var(--navbar-height); }

/* After */
.navbar {
  position: sticky;
  top: 0;
}
.hero { /* margin-top 제거 */ }
.page-content { min-height: 100vh; }
```

#### 효과
- navbar와 본문이 **동일한 콘텐츠 흐름(flow)** 에서 정렬
- `margin-top` 보정 코드 삭제 (자연스러운 문서 흐름)
- 스크롤 감지(`window.scrollY`) 정상 동작 유지
- subnav `position: sticky; top: navbar-height` 정상 동작 유지
