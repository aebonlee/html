## 2026-03-07 (금) — 교육신청 "자세히 보기" 상세 페이지 구현

### 작업 개요
교육신청(`#page=edu`) 페이지의 3개 과정(입문/중급/고급) 카드에 있던 "자세히 보기" 버튼에 실제 상세 페이지를 연결. 각 과정의 커리큘럼, 교육 기간, 대상, 선수 조건, 수료 역량 등 상세 정보를 한/영 이중 언어로 제공.

### 2. 과정 데이터 구조 확장 — `getEduCourses()`

**파일**: `index.html`

#### 기존 → 변경
기존 `renderEduView()` 내부에 인라인으로 선언되던 과정 데이터를 `getEduCourses()` 함수로 분리하고, 각 과정에 상세 정보 추가.

#### 추가된 데이터 필드
| 필드 | 설명 | 예시 (입문) |
|------|------|------------|
| `duration` | 교육 기간 | 4주 (주 2회, 회당 2시간) |
| `target` | 대상 수강자 | 프로그래밍 경험이 없는 입문자 |
| `prerequisite` | 선수 조건 | 없음 (컴퓨터 기본 사용 능력만 필요) |
| `outcome` | 수료 후 역량 | 간단한 웹 앱을 AI 도구로 독립 제작 |
| `curriculum[]` | 주차별 커리큘럼 | week, title, topics[] |

#### 과정별 커리큘럼 규모
| 과정 | 주차 | 총 토픽 수 |
|------|------|-----------|
| 입문 (바이브 코딩 기초) | 4주 | 16개 |
| 중급 (UI/UX 디자인 with AI) | 6주 | 24개 |
| 고급 (풀스택 AI 개발) | 7~8주 | 28개 |

### 3. 상세 페이지 렌더링 — `renderEduDetailView()`

**파일**: `index.html`

#### 페이지 구성
1. **뒤로 가기 버튼** — "교육과정 목록" / "Course List"
2. **과정 헤더** — 배지(레벨) + 제목 + 설명 (중앙 정렬)
3. **과정 정보 섹션** — 4개 항목 (기간/대상/선수조건/수료역량)
   - 아이콘 + 레이블 + 값 레이아웃
4. **커리큘럼 섹션** — 주차별 카드 그리드 (2열, 모바일 1열)
   - 주차 배지 (레벨 색상) + 제목 + 토픽 체크리스트
5. **연락처 카드** — 기존 교육 문의 정보 재사용

#### 과정별 레벨 색상
| 과정 | 색상 코드 | 용도 |
|------|----------|------|
| 입문 | `#059669` (Green) | 배지, 아이콘, 주차 뱃지 |
| 중급 | `#0046C8` (Blue) | 배지, 아이콘, 주차 뱃지 |
| 고급 | `#7C3AED` (Purple) | 배지, 아이콘, 주차 뱃지 |

### 5. URL 해시 라우팅 확장

**파일**: `index.html`

#### 변경 내용
- `updateHash()`: 교육 상세 보기 시 `#page=edu&course=0` 형태 해시 생성
- `handleHash()`: `course` 파라미터 파싱 → `selectedEduCourse` 복원

|-----|------|

| `#page=edu&course=0` | 입문 과정 상세 |
| `#page=edu&course=1` | 중급 과정 상세 |
| `#page=edu&course=2` | 고급 과정 상세 |
